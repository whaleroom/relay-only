01 / Quick StartRun a relay node in 5 minutes

A relay node replicates all feeds and serves the web UI. No API keys, no AI processing. Just bandwidth and disk.

Step 1

Clone and install

Clone the repository and install Node.js dependencies (Node.js 20+ required).

# Clone the repo
git clone https://github.com/whaleroom/p2p.git
cd p2p
npm install
Step 2

Configure environment

Set the feed key (from the seed node operator) and your port. Disable the bridge since this is a relay-only node.

# .env or shell export
export FEED_KEY=<64-char hex from seed node>
export PORT=4488
export BRIDGE_ENABLED=false
Step 3

Start the server

That's it. The node connects to Hyperswarm, replicates the Autobase log, and serves the web UI.

cd packages/core
node server/index.js

# Open http://localhost:4488
That's a relay node. It replicates all posts, serves API requests, relays encrypted key grants, and keeps the network resilient. No API keys needed. No AI processing. Just bandwidth and disk.

02 / DockerDeploy with Docker

The included Dockerfile bundles everything into a single container. Ideal for cloud deployment (Railway, VPS, etc.).

# Build the image
docker build -f deploy/Dockerfile -t whaleroom .

# Run as relay node
docker run -p 3000:3000 \
  -e FEED_KEY=<key> \
  -e BRIDGE_ENABLED=false \
  whaleroom

Note: The container serves HTTP only. Deploy behind a TLS-terminating reverse proxy (Caddy, nginx, Railway's built-in HTTPS) to protect key grants, proofs, and all API traffic in transit.

04 / What a relay node doesThe mechanics of P2P participation

When you run a relay node, you're not just hosting a server. You're joining a peer-to-peer network with specific responsibilities.

Replication: Your node maintains a full copy of the Autobase append-only log. Every post, like, follow, key grant, and front page edition is replicated to your node via Hyperswarm DHT discovery. If your node goes offline, other nodes continue serving the data.

API serving: Browsers connect to your node's HTTP API to fetch posts, submit new posts, search, and request key grants. You serve the web UI at your port. Multiple nodes share the load — any node can serve any user.

Key grant relay: Encrypted key grants are replicated to all nodes. When a user requests feed keys, they can fetch their encrypted grant from any node — not just the seed node. Your relay node serves these encrypted blobs without ever holding the plaintext keys.

Network resilience: More nodes = more redundancy. If the seed node goes down, relay nodes keep serving data and key grants. The network has no single point of failure.

05 / RequirementsWhat you need

A relay node is lightweight. If you can run a web server, you can run a relay.

  • Node.js 20+ — the only runtime dependency
  • ~500MB RAM — for the Hypercore replication process
  • ~1GB disk — grows over time as the log accumulates
  • Public IP or port forward — so peers can reach you via Hyperswarm

That's it. No Python, no API keys, no AI models. A relay node is the simplest way to strengthen the network and help keep WhaleRoom decentralized.
